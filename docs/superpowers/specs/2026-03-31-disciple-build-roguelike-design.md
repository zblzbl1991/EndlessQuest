# 弟子 Build 轻操作化设计稿

**日期**: 2026-03-31  
**状态**: 草案  
**产品方向**: 放置经营 + 自动化探险 + 弟子 Build 驱动策略

---

## 一、设计目标

本设计稿用于明确 EndlessQuest 在“弟子 build”上的产品方向：

- 保留 build 带来的策略差异与可复盘性
- 增加 Roguelike 式随机性，让每次探险都能长出不同解法
- 削减繁琐的手动配置，让玩家把注意力放在高价值决策上
- 让经营、管理、养成真正服务于“这次该派谁、赌什么流派、承担什么风险”

一句话定义：

**玩家经营的不是一批需要频繁手操配置的角色面板，而是一池会在随机事件中分化命运、在探险中长出不同 build 的弟子。**

---

## 二、核心判断

当前项目已经具备以下基础：

- 宗门经营底盘已成型
- 自动化秘境与战报系统已成型
- 战术预设、祝福、遗珍、路线分支已经具备 Roguelike 框架
- 弟子已有修行路线、命格、专长等 build 雏形

但当前 build 体验仍有两个主要问题：

1. build 的“表达”大于“决策”
玩家能看到很多标签和系统字段，但真正需要拍板的高价值决定还不够集中。

2. build 的“长期层”和“局内层”没有彻底分工
玩家既没有获得足够鲜明的长期弟子定位，也没有在单局中稳定感受到 build 成长的戏剧性。

因此，本次设计的核心不是增加更多 build 面板，而是：

**把 build 从“细碎配置”重构成“长期定向 + 局内生长”。**

---

## 三、设计原则

### 1. 少操作，不少决策

减少重复点击、反复换装、手动调槽位，但保留：

- 选谁上场
- 赌哪条成长分支
- 选什么局内流派
- 用什么宗门资源去培养什么人

### 2. 长期身份固定，局内结果多变

弟子需要先是“什么人”，然后才是“这局成了什么”。

- 长期层决定弟子的底色与成长潜力
- 局内层决定这一局具体成型的 build

### 3. 随机性服务于叙事与策略

随机不是为了噪音，而是为了制造：

- 预期外的组合
- 风险与收益的取舍
- 可以复盘的因果链
- “这个弟子这次打出了不同命运”的故事感

### 4. 经营要能改变随机池

宗门经营不能只是后台数值工厂，必须能影响：

- 招募到什么弟子
- 更容易出现什么 build
- 秘境中更容易刷出什么祝福、遗珍、事件

---

## 四、Build 的双层结构

弟子 build 拆成两层：

### A. 常驻 Build：弟子的长期身份

用于回答“这个弟子是什么人”。

包括：

- 根骨/资质偏向
- 专长
- 修行流派
- 命格与伤痕
- 长期成长倾向

特点：

- 更新频率低
- 不要求玩家频繁干预
- 应该强烈影响岗位价值、出战价值和可塑性

### B. 局内 Build：弟子本局的成型方向

用于回答“这个弟子这局成了什么”。

包括：

- 祝福
- 遗珍
- 路线分支
- 临时觉醒
- 局内事件附加状态

特点：

- 在一次秘境中快速成型
- 允许偏离长期身份
- 支撑每局不同体验

这两层必须明确分工：

- 常驻 build 决定随机池权重和成长上限
- 局内 build 决定即时玩法和局内爽点

---

## 五、应该砍掉的繁琐手动操作

为了让 build 更像 Roguelike，而不是传统养成页游，建议主动削减以下操作：

### 1. 弱化手动技能栏管理

不建议把技能配置做成高频手动操作。

调整方向：

- 技能主要由修行路线、命格、局内奇遇自动获得
- 玩家只在关键节点做少量“保留/替换”选择
- 平时通过战术预设影响技能释放倾向，而非逐个调技能逻辑

### 2. 弱化逐件装备微操

不建议要求玩家频繁逐格对比和替换装备。

调整方向：

- 装备更偏“词条方向”或“套装倾向”
- 强化“自动装备推荐”
- 玩家仅在稀有装备、特殊遗物、关键流派时介入

### 3. 弱化岗位反复拖拽

不建议让经营优化变成琐碎排班。

调整方向：

- 增加“推荐驻守”
- 增加“按专长自动派遣”
- 玩家重点决定的是培养谁、保留谁、让谁转岗，而不是每次手挪位置

### 4. 弱化逐层人工确认

自动化秘境方向应该继续坚持。

调整方向：

- 玩家在出发前做计划
- 系统在过程里自动结算
- 玩家通过战报和摘要回顾 build 结果

---

## 六、保留并强化的核心手动决策

玩家应该主要只做以下四类决策：

### 1. 选谁

决定这次出战的弟子组合。

关注点：

- 哪些人更适合这次目标秘境
- 谁适合冒险，谁更适合留守宗门
- 是否让高潜力弟子承受风险

### 2. 选方向

决定长期培养和这次出发的总体方向。

关注点：

- 宗门路线
- 本次战术预设
- 补给策略
- 是否押注某个 build 方向

### 3. 选分支

决定弟子在关键节点如何成长。

关注点：

- 突破时的成长岔路
- 局内 draft 的保留与替换
- 接受收益还是接受风险

### 4. 选投入

决定宗门资源优先投给谁。

关注点：

- 重点培养弟子
- 重点发展的宗门方向
- 是否为某种 build 改造经济结构

## 七、弟子常驻 Build 设计

## 7.1 根骨与初始偏向

每名弟子招募时带一个“底子”，决定初始倾向。

示例：

- 剑胚：暴击与先手潜力更高
- 丹心：恢复、辅助、炼丹相关权重更高
- 兽缘：宠物联动与奇遇收获更高
- 劫痕：高风险成长，容易留下负面痕迹但爆发潜力高
- 厚土：更偏防御与生存
- 灵窍：更容易领悟局内临时能力

设计目标：

- 让招募本身就包含 build 判断
- 让“这人值不值得养”成为经营决策

## 7.2 专长系统

专长应真正进入经营和探险逻辑，而不是只作为展示字段。

专长分为两类：

- 经营专长：炼丹、锻造、采矿、采药、参悟
- 探险专长：战斗、探路、福缘、统御

作用：

- 经营专长决定其留守宗门的价值
- 探险专长决定其出战和派遣价值

关键原则：

- 一个弟子不一定“强”，但可以“有用”
- 让留宗弟子和出战弟子都拥有清晰价值

## 7.3 修行路线

修行路线不应随机发牌，而应在关键节点提供选择。

建议在首次关键突破时，从 2 到 3 个方向里选择 1 个：

- 剑修：爆发、斩杀、先手
- 体修：承伤、反击、稳健推进
- 丹修：恢复、续航、资源利用
- 法修：群攻、属性联动、事件爆发
- 驭兽：宠物流、召应、联动收益

修行路线作用：

- 决定该弟子的祝福权重
- 决定局内更容易长成的 build
- 决定部分事件的特殊分支

## 7.4 命格与伤痕

命格是长期 build 的重要差异来源。

来源：

- 突破成功
- 渡劫失败
- 秘境事件
- 特殊招募

示例：

- 雷痕未消：更脆，但雷系与暴击收益更高
- 丹心稳固：恢复效果更高，突破更稳
- 心魔暗涌：高压时输出更强，但休整收益下降
- 见微知著：更容易刷到高质量局内 draft

设计目标：

- 让失败也能进入 build
- 让弟子成长路径更有命运感

---

## 八、局内 Build 设计

## 8.1 局内成长不再只看队伍，而要看“弟子”

当前秘境的祝福/遗珍更偏整队通用。

建议增加“弟子定向 build”：

- 某弟子获得专属觉醒
- 某弟子触发命格共鸣
- 某弟子获得本局专属形态强化

示例：

- 剑胚弟子可在局内分化为“快剑”“赌暴”“斩杀”
- 丹心弟子可分化为“续航”“保命”“药毒双修”
- 劫痕弟子可分化为“低血爆发”“渡劫反击”“雷伤连锁”

## 8.2 局内 Draft 结构

建议把局内 build 选择分为三层：

### 1. 祝福

高频、轻量、偏通用。

功能：

- 给基础属性和机制修正
- 快速塑造本局方向

### 2. 异变

中频、偏弟子个体。

功能：

- 绑定到某名弟子
- 产生更强 build 偏向

示例：

- 某弟子普通攻击附带雷痕
- 某弟子每损失 20% 血量提升暴击
- 某弟子击杀后给全队恢复

### 3. 遗珍

低频、高影响、可改规则。

功能：

- 打开新 build
- 改写风险收益结构

示例：

- 低血弟子不再容易撤退
- 所有恢复转化为护盾
- 宠物伤害改为由主人暴击率结算

## 8.3 路线与事件的 Build 倾向化

路线不应只代表风险高低，还要代表 build 方向。

建议将路线类型显性化：

- 求稳路：恢复、补给、防守 build
- 求战路：爆发、战斗成长、速攻 build
- 求财路：资源、掉落、经营转化 build
- 求变路：异变、命格、特殊事件 build

这样自动化探险即使不手操，也能有明显 build 分化。

---

## 九、宗门经营如何服务 Build

## 9.1 经营的核心作用：改变随机池

宗门经营最重要的不是单纯提高数值，而是改变未来 build 的出现概率。

可影响内容：

- 招募弟子的底子分布
- 秘境祝福池倾向
- 特殊事件出现概率
- 某类装备、丹药、遗珍更容易出现

## 9.2 宗门路线影响 Build 生态

宗门路线要成为“build 元环境”。

示例：

### 丹道宗

- 更容易招到丹心、稳健型弟子
- 恢复与续航类祝福权重提高
- 低风险 build 更强

### 剑道宗

- 更容易招到剑胚、爆发型弟子
- 斩杀、暴击、先手 build 权重提高
- 高风险高收益 build 更强

### 兽道宗

- 宠物流和协同 build 权重提高
- 探险里更容易遇到灵兽异变
- 出战弟子和灵宠之间形成独特 build

## 9.3 建筑的 Build 化

建筑不仅产资源，还应提供“build 倾向”。

示例：

- 炼丹房：提高恢复型局内选项质量
- 锻造坊：提高武器流、破甲流相关 build 权重
- 藏经阁：提高领悟、技能异变和路线分支质量
- 灵兽苑：提高宠物流与捕获相关 build 机会

---

## 十、玩家体验流程

新的弟子 build 体验应收束为以下流程：

### 1. 招募阶段

玩家关注：

- 这个弟子的底子是什么
- 适合留宗还是出战
- 值不值得长期培养

### 2. 培养阶段

玩家关注：

- 何时让其定向突破
- 让其走哪条修行路线
- 是否投入稀缺资源保他成型

### 3. 出发阶段

玩家关注：

- 这次派哪套阵容
- 赌什么 build 方向
- 用什么战术预设和补给策略

### 4. 局内阶段

系统自动推进，玩家主要通过摘要与战报获得反馈：

- 哪名弟子长成了什么
- 哪个随机事件改变了 build
- 这次为什么成功或失败

### 5. 回宗阶段

玩家关注：

- 哪些弟子留下了长期变化
- 本次收获是否支持下一次 build
- 要不要转向新的宗门经营方向

---

## 十一、战报与反馈要求

如果 build 轻操作化，战报就必须更强。

战报不只是展示结果，而要解释 build：

- 哪名弟子是本局核心
- 这名弟子吃到了什么关键异变
- 关键转折是什么事件
- 为什么这局失败或成功

建议新增的摘要维度：

- 本局核心弟子
- 本局成型流派
- 关键祝福/异变/遗珍
- 最大风险事件
- 本局宗门经营带来的关键加成

如果玩家看完战报能说出：

“这局是因为某个劫痕弟子赌低血暴击成了”

那 build 设计就是成功的。

---

## 十二、对现有项目的落地建议

按当前代码状态，建议分三步推进。

### 第一阶段：把弟子长期身份做实

优先级最高。

目标：

- 角色生成时真正 roll 专长
- 修行路线改为关键节点选择
- 命格效果进入正式成长与探险逻辑
- 招募页和弟子页能清楚表达“此人用途”

### 第二阶段：把局内 build 从整队通用改成弟子导向

目标：

- 祝福之外增加“弟子异变”
- 事件可绑定到个体弟子
- 局内 build 成果能在战报中明确归因到某位弟子

### 第三阶段：让经营影响随机池

目标：

- 建筑与宗门路线真正进入随机池计算
- 不同经营路线产生不同 build 生态
- 玩家能明显感受到“宗门方向不同，弟子命运不同”

---

## 十三、明确不做的方向

为保持产品聚焦，以下方向不作为当前 build 设计重点：

- 高频手动技能栏微操
- 高频逐件换装
- 复杂 AI 脚本编辑器
- 实时手操战斗
- 以面板数值堆叠为主的 build

这些内容即使实现，也应作为辅助层，而不是主乐趣来源。

---

## 十四、结论

弟子 build 的正确方向不是“做更多手动配置项”，而是：

**让弟子先拥有长期身份，再在每次秘境中长出不同命运。**

玩家应该体验到的是：

- 招募时判断价值
- 培养时赌方向
- 出发时做阵容和策略选择
- 回看战报时看到意料之外却可解释的 build 成型

这会让 EndlessQuest 的核心乐趣从“优化一个静态阵容”转向：

**经营一整个会不断分化、失控、惊喜、沉淀的弟子生态。**

这也更符合项目当前的放置经营 + 自动化探险方向。
