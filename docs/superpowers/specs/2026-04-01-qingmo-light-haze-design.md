# 清墨轻岚主题改造设计稿

**日期**: 2026-04-01  
**状态**: 待实现  
**主题**: 全局主题 / 导航壳层 / 日常页面表面  
**方向**: `B 清墨雾青` 为主，`D 湖青轻岚` 为辅

---

## 一、设计背景

当前 EndlessQuest 的视觉问题，不在于单一的褐色难看，而在于整套配色被同时推向了“暖、厚、旧”：

- `theme.css` 的背景、边框、强调色都偏暖褐
- `globals.css` 的水墨晕染也带明显赭色和旧纸感
- `Sidebar` / `TopBar` / `BottomNav` 的壳层继续放大了深褐、暖黄、旧木的印象

结果是：

- 古朴感有了
- 但“清新、淡雅”不足
- 水墨味更像旧卷轴，而不是清墨、晨雾、淡岚

用户明确更偏好预览中的两组方向：

- `B. 清墨雾青`
- `D. 湖青轻岚`

因此本轮不再继续纠结“要不要褐色”，而是直接采用合成方案：

**以 `B 清墨雾青` 作为整体基底，以 `D 湖青轻岚` 作为强调与清爽边缘。**

---

## 二、核心设计结论

### 2.1 最终主题命名

本轮正式主题命名为：

**`清墨轻岚`**

它的目标气质是：

- 像宣纸上刚落开的淡墨
- 带一点清晨山岚与水气
- 清新，但不冷硬
- 淡雅，但不虚弱

### 2.2 主题配方

本轮采用下面的组合关系：

- `底色 / 面色 / 中性色`
  以 `B 清墨雾青` 为主  
  负责“透气、干净、轻雾感”

- `强调色 / 激活态 / 主要按钮`
  从 `D 湖青轻岚` 借一点更利落的青蓝灰  
  负责“清爽、明确、收束”

- `成功 / 危险 / 稀有`
  继续保留语义差异，但整体饱和度同步降低，避免在新主题下显得突兀

### 2.3 明确不做

- 不做蓝白后台风
- 不做现代 SaaS 冷灰风
- 不做紫蓝渐变仙侠风
- 不做“只是把棕色改成蓝色”的浅替换

本轮要保留的，是水墨和宣纸感；要替换的，是“旧暖褐”的主导权。

---

## 三、配色系统设计

### 3.1 背景层

背景不再偏黄褐，而改成偏冷的宣纸白与雾灰青：

- 页面主底色：更接近淡灰宣纸
- 纸面层：更浅、更干净，接近白纸但不能发冷
- 晕染层：从赭石墨斑改成青灰、浅雾、淡水迹

这意味着用户第一眼看到的不会再是“旧卷”，而是“清纸上有墨痕”。

### 3.2 中性色

边框、说明字、分隔、禁用态全部同步降暖：

- 主文字：深墨灰，不用纯黑
- 次文字：灰蓝墨
- 三级说明：雾灰
- 边框：浅青灰纸边

这样做的目的不是“更科技”，而是让页面更轻、更通透。

### 3.3 强调色

强调色从原本的暖棕，转成偏青灰的冷墨强调：

- 主强调：青灰墨蓝
- hover：略亮但不发荧
- 选中态背景：很淡的湖青晕开
- 激活边线：清晰但克制

原则是：

- 按钮必须仍然一眼可见
- 但不再像赭石印泥那样厚重
- 要像清墨中的重点落笔

### 3.4 语义色

成功、危险、稀有保留意义，但需要二次校准：

- 成功：更偏青绿，不要太草绿
- 危险：保留红，但底色更淡、更纸化
- 稀有：现有紫系需要降饱和，避免在新主题里跳出来

---

## 四、壳层改造范围

### 4.1 Sidebar

当前问题：

- 深褐底太重
- 容易像木柜、旧匣、旧殿

本轮改造目标：

- 变成浅青灰墨栏，接近“题签侧卷”
- 保留层次，但大幅减轻压迫感
- `seal` 仍可保留，但不再是深赭印章，而是更冷静的墨印青章

### 4.2 TopBar

当前问题：

- 暖米黄太多
- 像旧纸眉条，而不够清

本轮改造目标：

- 改成更轻的雾纸感顶部
- 资源 pill 保持存在，但像淡墨注记，不像暖色标签

### 4.3 BottomNav

当前问题：

- 目前与桌面侧栏虽已统一结构，但气质仍偏暖

本轮改造目标：

- 与新 Sidebar 使用同一套青灰清墨语言
- 激活态更像水痕晕开的题签，而不是暖色高亮

---

## 五、页面表面改造范围

### 5.1 Hero / Summary / Card

当前 hero、summary、card 虽然层级正确，但很多地方仍带暖棕底色。

本轮要求：

- hero 改成“淡墨洗开”的纸面高光
- summary 卡改成轻雾边界
- 普通卡片保持白纸感，但边界和阴影要更轻、更冷

### 5.2 主按钮与次按钮

主按钮必须仍然清晰可见，符合项目的“可操作即可见”，但视觉语气要更新：

- 主按钮：青灰墨蓝实体
- 次按钮：淡湖青边界或浅墨文字按钮
- 禁用态：灰雾纸色

### 5.3 高光时刻

已经做过的高光页不推翻，只调整色相：

- 高光仍然要浓墨重彩
- 但不再靠赭石与旧金感
- 改为更冷静的墨涌、水光、清印式对比

## 六、实施顺序

### 第一层：主题 token

先改：

- `src/styles/theme.css`
- `src/styles/globals.css`

目标是先把全局底色、文字、边框、accent 和晕染语气改正。

### 第二层：壳层

再改：

- `Sidebar`
- `TopBar`
- `BottomNav`

目标是最先消除“深褐外壳”的旧感。

### 第三层：日常页面表面

最后统一：

- hero
- summary card
- section card
- 常用按钮

重点页面优先顺序：

1. `SectPage`
2. `AdventurePage`
3. `BuildingsPage`
4. `CharactersPage`

---

## 七、验收标准

本轮完成后，应满足以下判断：

1. 第一眼不再觉得“偏褐、偏旧、偏闷”
2. 整体明显更清新、更透气，但仍然是仙侠水墨而不是现代后台
3. 壳层与页面表面使用同一套青灰清墨语言，不再出现暖褐残留主导
4. 主按钮依然清楚，交互可见性不下降
5. 高光时刻仍然有反差，只是色相从暖褐系转向清墨系

如果做完后用户的第一反应是“更轻、更净、更像清晨水墨”，那这轮就成功了。
