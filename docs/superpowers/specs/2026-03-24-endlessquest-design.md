# EndlessQuest - 仙侠放置 Roguelike 游戏 设计文档

## 概述

东方仙侠题材的放置类网页游戏，融合养成、经营与冒险战斗三大系统。冒险战斗采用事件驱动型 Roguelike 模式。角色自动战斗，玩家专注于修炼成长、宗门经营和策略决策。

- **平台**：Web (HTML5)
- **渲染**：DOM + CSS
- **语言**：TypeScript

## 技术栈

- **框架**：React 18 + TypeScript
- **构建**：Vite
- **状态管理**：Zustand（持久化到 localStorage）
- **样式**：CSS Modules + CSS Variables（仙侠古风主题）
- **路由**：React Router v6
- **数据存储**：localStorage + IndexedDB（存档系统）
- **动画**：CSS Animations + Framer Motion（战斗特效）

## 页面导航

底部导航栏：大殿 → 修炼 → 宗门 → 冒险 → 背包

- **大殿**：角色概览、资源概览、快捷修炼入口、宗门动态
- **修炼**：境界突破、功法学习/装备、属性查看
- **宗门**：建筑管理、资源生产、弟子管理
- **冒险**：秘境选择、组队编成、挂机/手动模式切换
- **背包**：物品管理、装备穿戴/强化/精炼
- **交易**：坊市商店、拍卖行、NPC 交易（从宗门页或背包页进入）

## 核心游戏循环

```
宗门经营（建筑/资源/弟子）
    → 角色养成（修炼/装备/功法）  ← 交叉互补
    → Roguelike 冒险（秘境/事件/战斗）
    → 资源收获（战利品/灵材/经验）
    → 回馈经营和养成 → 循环
```

经营产出（装备/丹药/弟子）提升冒险战力，冒险产出（材料/功法/图纸）回馈经营升级。两个体系各自有独立进度线，不互为硬性解锁前提。

## 一、修炼养成系统

### 1.1 境界体系

大境界分 4 个小层级：初期 → 中期 → 后期 → 圆满

| 大境界 | 小层级 | 修为需求（每个小层） | 突破额外条件 | 解锁内容 |
|--------|--------|----------------------|---------------|----------|
| 炼气期 | 初→中→后→圆 | 100, 300, 600, 1000 | 无 | 基础功法、灵品装备掉落 |
| 筑基期 | 初→中→后→圆 | 2000, 4000, 7000, 11000 | 筑基丹 ×1 | 秘境解锁（落云洞） |
| 金丹期 | 初→中→后→圆 | 18000, 28000, 40000, 55000 | 金丹劫(敌人战力=自身×0.8) | 灵宠系统 |
| 元婴期 | 初→中→后→圆 | 75000, 100000, 130000, 170000 | 元婴劫(敌人战力=自身×1.0) | 弟子系统（经营线） |
| 化神期 | 初→中→后→圆 | 220000, 280000, 350000, 450000 | 天劫(3阶段战斗，战力逐级×1.0/1.2/1.5) | 高级功法、飞剑 |
| 渡劫飞升 | 最终 | 600000 | 全建筑满级(建筑线) | 游戏通关（终局） |

**境界属性成长：** 每突破一个大境界，全属性 = 上一境界圆满值 × 1.8。每个小层级提供小幅属性增长（约大境界的 15%）。

**天劫战斗规则：** 天劫为 PvE 战斗，敌人属性按比例缩放（见上表倍率）。击败即突破成功，失败可无限重试，无惩罚。

### 1.2 角色属性

**基础属性：**
- HP（生命）：初始 100，每境界大幅增长
- ATK（攻击）：初始 15，影响技能伤害
- DEF（防御）：初始 8，减免受到伤害
- SPD（速度）：初始 10，决定战斗出手顺序（每回合按 SPD 排序，每人行动一次）
- CRIT（暴击率）：初始 5%，上限 75%
- CRIT_DMG（暴击伤害倍率）：初始 150%

**修仙属性：**
- 修为：修炼累积，突破消耗（见 1.1 修为需求表）
- 灵力：技能消耗资源，战斗中每回合回复 10 点，最大值 = 50 + 神识功法加成
- 悟性：每点悟性 = 功法升级时间缩短 2%，初始 10
- 灵根：每点灵根 = 修为获取速率 +2%，初始 10
- 气运：每点气运 = 稀有掉落率 +1%，随机事件正面结果概率 +2%，初始 5

**属性来源：** 基础属性 = 境界基础值 + 装备加成 + 功法加成。修仙属性 = 初始值 + 灵根(固定) + 功法加成 + 临时增益。

### 1.3 功法/技能系统

**功法（被动）：**
- 分类：心法、体修、神识
- 永久提升角色属性
- 通过修炼时间/秘籍解锁，可升级：初级 → 中级 → 高级 → 顶级
- 装备栏位：心法(1) + 体修(1) + 神识(1)

**技能（主动）：**
- 分类：攻击、防御、辅助、终极
- 战斗中自动释放，有冷却时间和灵力消耗
- 伤害 = ATK × 技能倍率 × 属性克制
- 装备栏位：主动技能(4) + 终极技能(1)

**灵力经济：**
- 最大灵力 = 50 + 神识功法等级 × 10（初始 50）
- 每回合回复 10 点灵力
- 主动技能消耗：10~25 灵力（按技能等级）
- 终极技能消耗：30~50 灵力（按技能等级）
- 灵力不足时跳过该技能，使用普攻

### 1.4 装备系统

**部位（9个）：** 头冠、道袍、护腕、腰带、鞋子、武器、饰品(2)、法宝

**品质等级：** 凡品(灰) → 灵品(蓝) → 仙品(紫) → 神品(金) → 混沌(橙)

**强化方式：**
- 强化：+1 ~ +15，消耗灵石和矿材。成功率：+1~+5 100%，+6~+10 90%~70%（每级-4%），+11~+15 60%~20%（每级-10%）。失败不降级，仅浪费材料
- 精炼：附加 1 条随机属性，可反复精炼覆盖（消耗灵石+精炼石）
- 套装：同套装装备 2/4/6 件激活对应效果（具体套装内容在 data/items.ts 中定义）

**背包系统：**
- 初始背包 50 格，可通过仙玉扩容（+10格/次，上限 200 格）
- 资源（灵石/灵气/灵草/矿材等）为数值型，不占背包格数，有独立仓库
- 背包满时新物品无法拾取，需先清理；挂机模式下自动卖出最低品质物品腾出空间

### 1.5 灵宠/弟子战斗编队

队伍组成：主角(必上阵) + 灵宠位(2) + 弟子位(1)，最多 4 个单位

- 灵宠：通过秘境捕捉/商店购买，独立养成线，编入队伍协同战斗
- 弟子：通过宗门招募，修炼/任务成长，可派遣或编入战斗

### 1.6 灵宠系统

**灵宠品质：** 凡 → 灵 → 仙 → 神（与装备品质色对应）

**灵宠属性：** HP、ATK、DEF、SPD，初始值按品质决定（灵品约为角色同境界的 40%，每升一品质 ×1.5）

**灵宠技能：** 每只灵宠有 1 个 innate 技能（固定）+ 2 个可装备技能位。技能 CD 独立计算。

**灵宠捕捉：** 秘境中"随机事件"可能遇到灵宠。捕捉概率 = 基础 30% + 气运 × 2%，品质越低概率越高。失败后灵宠逃走，不可重复捕捉同一只。

**灵宠培养：** 喂养兽魂提升等级，每级全属性 +3%。最大等级 = 品质 × 20（凡品 Lv20，灵品 Lv40，以此类推）。

**灵兽数量上限：** 最多拥有 12 只灵宠（可通过仙玉扩容）。

## 二、宗门经营系统

### 解锁原则

经营线内部驱动：建筑升级条件来自**建筑等级**和**弟子品质**。
冒险线内部驱动：秘境/境界条件来自**角色境界**和**战力**。
两个体系通过资源交叉互补，不互为硬性解锁前提。

### 2.1 建筑体系

| 建筑 | 功能 | 最大等级 | 升级效果 | 解锁条件（经营线） |
|------|------|---------|----------|-------------------|
| 宗门大殿 | 核心建筑 | Lv10 | 解锁其他建筑等级上限 | 初始 |
| 灵田 | 产出灵草/灵材 | Lv10 | 产出速度 + 品种 | 大殿 Lv1 |
| 坊市 | NPC 商店 | Lv8 | 商品品质 + 刷新 | 大殿 Lv1 |
| 丹炉 | 炼制丹药 | Lv8 | 丹药品种 + 成功率 | 大殿 Lv2 + 灵田 Lv2 |
| 炼器坊 | 锻造/强化装备 | Lv8 | 强化上限 + 品质 | 大殿 Lv2 + 拥有凡品弟子×2 |
| 藏经阁 | 学习功法 | Lv8 | 可学功法等级 | 大殿 Lv3 + 拥有灵品弟子×1 |
| 聚仙台 | 招募弟子 | Lv6 | 弟子品质上限 | 大殿 Lv3 + 炼器坊 Lv2 |
| 传功殿 | 弟子修炼/派遣 | Lv6 | 派遣效率 + 任务类型 | 大殿 Lv4 + 拥有灵品弟子×3 |

### 2.2 资源系统

**基础资源：**
- 灵石（通用货币，建筑/升级消耗）
- 灵气（放置自动产出，修炼消耗）
- 灵草（灵田产出，炼丹材料）
- 矿材（采集/冒险获取，炼器材料）

**稀有资源：**
- 仙玉（高级货币，稀有兑换）
- 功法残页（合成完整功法）
- 天地异宝（突破/飞升必需）
- 兽魂（灵宠培养材料）

**资源产出公式：** 每秒产出 = 基础值 × 建筑等级倍率 × 功法加成 × 弟子加成

**零资源保护：** 灵田始终产出灵气（基础 1/s），保证玩家永远不会完全卡死。即使灵石为 0，修炼仍可继续（只消耗灵气）。

### 2.3 弟子系统

**生命周期：** 招募（聚仙台）→ 修炼（传功殿）→ 派遣（执行任务）→ 出战（编入队伍）

**弟子属性：**
- 品质：凡 → 灵 → 仙 → 神
- 天赋：影响修炼速度
- 忠诚：影响派遣效率
- 专属功法位(2) + 技能位(3)

**弟子死亡规则：** 弟子在战斗中败亡后进入"重伤"状态，需在传功殿恢复（恢复时间 = 60s × 品质倍数）。重伤期间不可出战和派遣。弟子**不会永久死亡**。建筑解锁条件按"历史最高拥有"计算，不会因弟子重伤而失去已解锁功能。

**弟子等级上限：** 凡品 Lv30，灵品 Lv50，仙品 Lv70，神品 Lv99

**派遣任务类型：** 采集（灵草/矿材）、探索（发现秘境/宝物）、交易（出售物品获取灵石）、讨伐（自动打怪获取战利品）

### 2.4 交易系统

**NPC 商店（坊市）：** 固定商品 + 每日刷新稀有品。消耗灵石/仙玉购买基础物资、丹药、功法残页。

**拍卖行：** 珍稀度等级由拥有的最高品质弟子决定：

| 拍卖等级 | 解锁条件 | 可拍卖物品 |
|----------|----------|-----------|
| 凡品拍卖 | 坊市 Lv1（默认） | 凡品装备/基础丹药 |
| 灵品拍卖 | 拥有灵品弟子 ≥ 1 | 灵品装备/中级丹药/功法残页 |
| 仙品拍卖 | 拥有仙品弟子 ≥ 1 | 仙品装备/高级丹药/完整功法 |
| 神品拍卖 | 拥有神品弟子 ≥ 1 | 神品装备/天地异宝/上古功法 |

**NPC 交易：** 不同 NPC 收藏不同品类物品，出售对应品类可获溢价。部分特殊 NPC 只在特定条件下出现。

## 三、Roguelike 冒险系统

### 3.1 秘境总览

秘境流程：选择秘境 → 组队编成 → 进入秘境 → 事件探索 → Boss 战 → 结算奖励

每次进入秘境是一次独立的 Roguelike 流程，失败/通关后结算奖励。

**秘境运行规则：**
- 关闭浏览器时秘境进度保留（存入 IndexedDB），重新打开可继续
- 手动模式可随时撤退，保留当前层及之前的奖励（按层数比例计算，如 3/5 层 = 60% 奖励）
- 挂机模式死亡后自动结算已通过层数的奖励，从第 1 层重新开始
- 通关后获得全部奖励 + 通关奖励加成

### 3.2 秘境等级

解锁条件全部基于冒险线（角色境界），由冒险体系内部驱动。

| 秘境名称 | 层数 | 事件数/层 | 解锁条件 | 主要产出 |
|----------|------|-----------|----------|----------|
| 灵草谷 | 5 | 3 | 初始开放 | 灵草/凡品装备 |
| 落云洞 | 8 | 3 | 炼气圆满 | 矿材/灵品装备 |
| 血魔渊 | 10 | 4 | 筑基圆满 | 功法残页/丹方 |
| 龙骨荒原 | 12 | 4 | 金丹圆满 | 仙品装备/灵宠蛋 |
| 九幽炼狱 | 15 | 5 | 元婴圆满 | 天地异宝/上古功法 |
| 天劫秘境 | 20 | 5 | 化神圆满 | 飞升材料 |

### 3.3 事件系统

每层 3~5 个事件节点，玩家选择前进方向。

**事件类型分布：**
- 战斗事件（40%）：消灭普通怪/精英怪/小 Boss
- 随机事件（25%）：仙缘奇遇/陷阱/宝箱，有分支选择
- 事件商店（15%）：用局内货币购买增益
- 休息点（10%）：回复 HP，选择一个临时增益
- Boss 房（10%）：每层最终节点

**局内成长（本次秘境内生效）：**
- 临时增益：属性提升、技能强化、特殊效果（吸血/反弹/闪避）
- 功法选择：火系攻击（灼烧 DOT）、冰系控制（冻结）、雷系爆发（闪电链 AOE）、治愈系（战斗回血）
- 局内货币：灵玉碎片（仅本次秘境使用，结算清零）

**属性克制（五行）：** 火克冰、冰克雷、雷克火。克制时伤害 ×1.5，被克制时伤害 ×0.75。敌人和技能均带有五行属性标签。治愈系为中性，不受克制影响。

**层数缩放：** 每深入 1 层，敌人全属性 = 基础值 × (1 + 0.08 × 层数)。奖励品质随层数提升（高层更可能掉落高品质物品）。

### 3.4 路线选择

每层出现 3 条分支路线供玩家选择：
- **战斗路线**：多个战斗 → Boss，风险高奖励高
- **探索路线**：事件/宝箱/休息混合，风险中收益均衡
- **安稳路线**：休息/商店/少量战斗，风险低奖励少

### 3.5 自动战斗系统

战斗流程：回合开始 → 速度排序 → 释放技能 → 伤害结算 → 检查胜负

**技能释放逻辑：**
1. 优先释放 CD 好的终极技能
2. 其次释放主动技能（按预设优先级）
3. 所有技能 CD 时普攻
4. 灵力不足时跳过技能

**伤害公式：**
- 伤害 = ATK × 技能倍率 × 属性克制
- 实际伤害 = MAX(伤害 - DEF × 0.5, 1)
- 暴击时 × (1 + CRIT_DMG)
- 放置模式下每回合间隔 1.5 秒

### 3.6 放置与手动模式

**挂机模式：**
- 自动选择安稳路线推进（无安稳路线时选探索路线）
- 自动战斗，HP < 30% 时自动使用恢复丹药
- 死亡后自动结算已通过层数奖励，从第 1 层重新开始
- 适合离线/挂机时使用

**手动模式：**
- 手动选择路线和事件分支
- 手动选择临时功法和增益
- 战斗自动进行，可随时撤退
- 收益比挂机高 1.5~2 倍

## 四、存档系统

### 4.1 数据持久化

**localStorage（频繁读写）：**
- 游戏设置（音效/画质等）
- 最后在线时间戳
- 放置引擎快照（用于离线计算）
- 自动存档触发标记

**IndexedDB（结构化大数据）：**
- 角色完整属性
- 背包/装备数据
- 宗门建筑/弟子数据
- 冒险记录/秘境通关历史

### 4.2 存档规则

- 自动存档：每 30 秒 + 关闭页面前
- 存档槽位：3 个，可切换
- 导出/导入：支持 JSON 文件备份和迁移

### 4.3 存档安全

- 存档数据包含版本号（version 字段），版本更新时自动迁移（migration 函数）
- 导入时校验数据 schema，无效数据拒绝导入并提示
- 战斗进行中暂停自动存档，战斗结束后立即保存
- 浏览器清空数据导致存档丢失的，可通过之前导出的 JSON 恢复

## 六、放置引擎

### 6.1 在线模式

- setInterval 每 1 秒 tick
- 计算资源产出（灵气/灵草/矿材）
- 累积修为（基础值 × 功法加成）
- 更新弟子派遣倒计时
- 更新丹炉/炼器进度
- 刷新 UI 数据

### 6.2 离线模式

- 记录最后在线时间戳
- 登录时计算离线时长
- 离线收益 = 在线收益 × 75%，上限 24 小时
- 弹出离线收益面板
- 挂机冒险按安全路线模拟
