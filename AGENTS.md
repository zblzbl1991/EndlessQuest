# Project: EndlessQuest

挂机修仙宗门经营网页游戏。当前设计上下文见 `.impeccable.md`，原始总设计稿归档于 `docs/archive/superpowers/specs/2026-03-24-endlessquest-design.md`，本轮布局/风格审查见 `docs/superpowers/specs/2026-04-02-layout-style-review.md`，挂机修仙方向重定稿见 `docs/superpowers/specs/2026-04-10-idle-cultivation-product-realignment.md`。

## 产品方向更新（2026-04-10）

- 主方向统一为 **挂机修仙**，不是高频手操的肉鸽爬塔。
- 玩家主要负责定宗门方向、弟子分工、挂机远征模板和关键突破 / 飞升时机。
- 秘境、随机事件、祝福、遗物保留，但服务于自动远征与长期经营，不再作为日常主操作负担。
- 失败默认应表现为伤病、效率下降、收益回落、产线波动，而不是常态化永久死亡。
- 离线报告、宗门变化、瓶颈诊断和自动化策略优先级高于局内操作层。

## 设计上下文

### 用户
仙侠挂机 / 放置游戏爱好者。熟悉修仙放置类手游的低频上线与长期积累节奏，能接受随机性，但不希望被高频微操和高惩罚持续打断。主要在碎片时间和晚间放松时段游玩。

### 品牌气质
**古朴、禅意、有力量感**

### 美术方向
浅色水墨风格 + 仙侠手游 UI 布局。日常壳层以 `清墨轻岚` 为基底：冷宣纸白、雾青灰、浅墨蓝构成清淡素雅的日常界面；赭石 / 印泥色只保留给关键主操作、完成反馈和里程碑高光。避免 MMO 式重度 UI（太多按钮/弹窗/数字堆砌/满屏发光特效）。移动端底部标签栏 + 桌面端左侧侧边栏，CSS 响应式切换。

### 响应式策略
- **断点**：640px（移动/平板）、1024px（平板/桌面）
- **移动端（< 640px）**：底部标签导航、单列布局、底部抽屉弹窗、44px 最小触控区域
- **平板端（640-1023px）**：继续使用底部标签导航，但内容宽度、摘要区和列表组织必须明显强于手机，至少进入稳定的双列或主次并排结构
- **桌面端（≥ 1024px）**：左侧固定侧边栏导航、多列网格、面板并排、居中弹窗；宽屏空间优先用于持续可见的宗门上下文、摘要和并排信息，而不是简单居中放大移动端布局
- **参考风格**：修仙手游 PC 模拟器的原生化适配

### 设计原则

1. **留白即呼吸** — 水墨画的灵魂在于留白。UI 通过充足的间距和留白让界面"透气"。信息密度适度，重点是让玩家快速读懂宗门变化，而不是一次展示太多。
2. **水墨晕染，不张扬** — 视觉效果以水墨质感为主（渐变、晕染、淡入淡出）。日常界面保持清淡素雅，避免霓虹发光、厚重描边、过度饱和的颜色。
3. **关键时刻，浓墨重彩** — 日常界面保持禅意和克制，但境界突破、通关秘境、获得稀有物品等成就时刻必须有强烈的视觉反馈。反差越大，冲击越强。
4. **信息层级如山水远近** — 像山水画有近景中景远景，UI 有清晰的层级关系。最重要的信息最近最清晰，次要信息渐远渐淡。
5. **少操作，多配置** — 主要交互应以设置规则、切换模板、决定优先级为主，而不是高频点流程。
6. **移动优先，逐级增强** — 基础布局以移动端为起点，通过 `min-width` 媒体查询逐步增强桌面端体验。移动端底部导航，桌面端侧边栏导航，CSS 响应式切换。
7. **宽屏不浪费** — 桌面端不简单居中放大移动端布局，而是重新组织信息：列表变多列网格、面板可并排、详情可侧滑展开。保持禅意克制，不因空间充裕而堆砌信息。
8. **可操作即可见** — 玩家可执行的关键动作（调整策略、切换模板、突破、升级等）必须有明确的主强调色实心按钮提示，不可操作时切换为灰底灰字禁用态。
9. **平板不是放大的手机** — `640-1023px` 区间必须被单独设计，不能继续沿用 `max-width: 480px` 的手机容器和纯单列节奏。
10. **日常清墨，高光落印** — 日常页面保持清墨轻岚的呼吸感，关键时刻再用更重的印泥 / 浓墨反差抬高仪式感，不能让所有场景都停留在同一强度。
