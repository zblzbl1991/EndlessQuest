## 当前设计基线（2026-04-10）

### 项目定位
`EndlessQuest` 的主方向统一为：**挂机修仙宗门经营游戏**。  
秘境、随机事件、祝福与遗物依然保留，但它们服务于挂机远征与长期经营，不再作为高频手动肉鸽操作的主体验。

当前总设计稿归档于 `docs/archive/superpowers/specs/2026-03-24-endlessquest-design.md`。  
布局 / 风格审查见 `docs/superpowers/specs/2026-04-02-layout-style-review.md`。  
挂机修仙方向重定稿见 `docs/superpowers/specs/2026-04-10-idle-cultivation-product-realignment.md`。

### 目标用户
仙侠挂机 / 放置游戏爱好者，偏好低频上线、长期积累、角色养成、宗门经营与离线成长。  
他们接受随机性，但不希望被高频手操和高惩罚打断节奏。

### 核心体验
玩家负责定方向，宗门负责自己成长。

玩家的主要行为应收束为：
- 选择宗门方针
- 调整弟子分工
- 配置挂机远征模板
- 决定关键突破 / 飞升时机

玩家不应被迫频繁执行：
- 局内逐层路线选择
- 高频战斗微操
- 重复收菜式点按
- 为避免浪费而高频上线

### 产品支柱
1. **低操作，高反馈**  
   每次上线在 30-90 秒内完成高价值决策，并看到明确变化。

2. **离线也在生长**  
   下线期间宗门应持续产出资源、推进闭关、执行远征、触发事件、产生损耗与收获。

3. **配置优于手操**  
   爽点来自配置正确后系统自转得越来越顺，而不是依靠手动临场操作翻盘。

4. **失败是节奏波动，不是体验中断**  
   挂机主循环中的失败默认表现为伤病、收益下降、产线停顿、秘境回退；永久损失只留给极端高风险玩法。

5. **阶段跃迁要强烈**  
   突破、主殿升级、解锁新模板、稳定新秘境、飞升轮回，都是挂机游戏的核心高潮点，必须显著高于日常态。

### 品牌气质
三个关键词：**古朴、禅意、有力量感**

- **古朴**：整体视觉扎根于传统东方美学，强调水墨、宣纸、书法等元素
- **禅意**：大量留白、舒缓节奏、不喧闹的界面
- **有力量感**：平时克制，关键节点必须有浓墨重彩的压迫感和仪式感

### 美术方向
- **视觉方向**：浅色水墨风格 + 仙侠手游 UI 布局
- **日常基线**：`清墨轻岚`
- **高光落点**：赭石 / 印泥色只留给主操作、完成反馈、突破、飞升、稀有掉落与里程碑
- **反参考**：避免 MMO 式重度 UI、通用后台式密集面板、过强任务清单指挥感

### 页面定位
- **宗门页**：挂机总控台，展示宗门变化、瓶颈、挂机策略与离线预期
- **弟子页**：人才策略页，重点是分工、培养梯队、闭关目标、恢复调度
- **建筑页**：生产调度台，重点是产线效率、自动规则、溢出保护、升级建议
- **秘境页**：自动远征配置页，重点是模板、风险倾向、收益偏好、结果回看
- **战报页**：自动调参反馈页，重点是“为什么赚 / 为什么亏 / 下一步怎么改”
- **离线报告页**：情绪结算页，重点是宗门在离线期间发生了什么

### 响应式策略
- **断点**：`640px`（移动端上限 / 平板起点）、`1024px`（平板上限 / 桌面起点）
- **移动端（< 640px）**：底部标签导航、单列布局、底部抽屉弹窗、44px 最小触控区域
- **平板端（640-1023px）**：继续使用底部标签导航，但内容宽度、摘要区和列表组织必须明显强于手机，至少进入稳定的双列或主次并排结构
- **桌面端（≥ 1024px）**：左侧固定侧边栏导航、多列网格、面板并排、居中弹窗；宽屏空间优先用于持续可见的宗门上下文、摘要和并排信息，而不是简单居中放大移动端布局
- **参考风格**：修仙手游 PC 模拟器的原生化适配

### 设计原则
1. **留白即呼吸**  
   UI 不是把所有信息塞满，而是让玩家在短时间内抓到最重要的变化与风险。

2. **少操作，多配置**  
   主要交互应以“设置规则、切换模板、决定优先级”为主，而不是频繁点击流程。

3. **日常清墨，高光落印**  
   日常界面轻、静、透；关键时刻再拉高反差和仪式感。

4. **少指挥，多呈现**  
   界面呈现宗门状态、瓶颈、波动和风险，不做强任务清单。

5. **可操作即可见**  
   可执行的关键动作必须一眼可见，并显著强于普通信息卡片。

6. **平板不是放大的手机**  
   `640-1023px` 必须有单独的信息编排，不能只是手机页面居中放大。

7. **宽屏不浪费**  
   桌面端优先用于持续可见的宗门上下文、摘要、并排信息和挂机控制。

8. **失败不能打断舒适节奏**  
   失败应更多表现为收益、效率、恢复与调度压力，而不是频繁永久损失。

9. **离线反馈优先展示事件，而非纯数字**  
   玩家回来首先要看到“发生了什么”，其次才是“加了多少”。

10. **挂机乐趣来自长期滚雪球**  
    任何新增系统，都应优先服务于长期经营、自转效率与阶段跃迁，而非短时手操刺激。
