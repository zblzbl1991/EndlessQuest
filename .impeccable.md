## Design Context

### Users
仙侠放置游戏爱好者与 Roguelike 策略玩家的交集人群。他们熟悉修仙放置/挂机类手游（如一念逍遥、修仙家族模拟器），也喜欢以撒、杀戮尖塔等 Roguelike 的策略深度。主要在碎片时间（通勤、午休）和晚间放松时段游玩，既享受挂机看着数字增长的满足感，也享受 Roguelike 中做策略选择时的投入感。

### Brand Personality
三个关键词：**古朴、禅意、有力量感**

- **古朴**：整体视觉 rooted 在传统东方美学，水墨、宣纸、书法元素
- **禅意**：大量留白、舒缓节奏、不喧闹的界面
- **有力量感**：境界突破、战斗胜利等关键时刻有强烈的视觉冲击和成就反馈

### Aesthetic Direction
- **视觉方向**：浅色水墨风格 + 仙侠手游 UI 布局
- **参考**：修仙放置类手游的 UI 信息密度（如一念逍遥的按钮布局和信息层级），但以浅色水墨为基底
- **反参考**：避免 MMO 式重度 UI（太多按钮/弹窗/数字堆砌/满屏发光特效）。界面要克制，信息层级清晰但不拥挤
- **主题**：Light mode，宣纸色为底，水墨晕染为装饰，赭石色为强调
- **布局**：移动端底部标签栏 + 桌面端左侧侧边栏，CSS 响应式切换，无需 JS 检测设备类型

### Responsive Strategy

移动端与 Web 端双端适配。移动端为主（核心体验），Web 端充分利用更宽屏幕提升信息密度和操作效率。

- **断点**：`640px`（移动端上限 / 平板起点）、`1024px`（平板上限 / 桌面起点）
- **移动端（< 640px）**：底部标签导航、单列布局、底部抽屉弹窗、44px 最小触控区域
- **平板端（640-1023px）**：底部标签导航、适当增加列数（2-3列网格）、弹窗居中
- **桌面端（≥ 1024px）**：左侧固定侧边栏导航（收起时仅图标，展开时图标+文字）、多列网格布局、面板可并排、弹窗居中对话框
- **导航切换**：桌面侧边栏与移动端底部导航通过同一组件 + CSS 响应式切换，不依赖 JS 检测
- **参考风格**：修仙手游 PC 模拟器体验的原生化适配（非简单居中模拟）

### Design Principles

1. **留白即呼吸（White Space as Breath）**
   水墨画的灵魂在于留白。UI 不是把所有信息塞满屏幕，而是通过充足的间距和留白让界面"透气"。每个面板、每个区块之间要有呼吸感。信息密度适度，宁可多一层点击也不要一次展示太多。

2. **水墨晕染，不张扬（Ink, Not Neon）**
   视觉效果以水墨质感为主——渐变、晕染、淡入淡出。特效（战斗、突破）可以用浓墨泼洒，但日常界面保持清淡素雅。避免霓虹发光、厚重的描边、过度饱和的颜色。

3. **关键时刻，浓墨重彩（Dramatic at Peaks）**
   日常界面保持禅意和克制，但境界突破、通关秘境、获得稀有物品等成就时刻必须有强烈的视觉反馈。用浓墨泼溅、金色光效、大幅动画来制造仪式感和成就感。反差越大，冲击越强。

4. **信息层级如山水远近（Clear Visual Hierarchy）**
   像山水画有近景中景远景一样，UI 有清晰的层级关系。最重要的信息（角色状态、当前操作）最近最清晰，次要信息（资源、进度）渐远渐淡。玩家一眼就能看到重点，不会被均匀的信息密度淹没。

5. **移动优先，逐级增强（Mobile-First, Progressive Enhancement）**
   基础布局以移动端为起点，通过 `min-width` 媒体查询逐步增强桌面端体验。移动端保持 44px 最小触控区域和底部导航；桌面端切换为侧边栏导航、多列网格、面板并排，充分利用屏幕宽度。同一份代码、同一套设计语言，不同断点下自然适配。

6. **宽屏不浪费（Use Width Wisely on Desktop）**
   桌面端不简单居中放大移动端布局，而是根据额外空间重新组织信息：列表变多列网格、详情面板可侧滑展开、相关面板可并排显示。但始终保持水墨禅意的克制风格，不因空间充裕而堆砌信息。

7. **可操作即可见（Affordance Must Be Visible）**
   当玩家可以对某物执行操作时（建造、升级、锻造、参悟等），必须有明确的按钮视觉提示。主操作按钮统一使用赭石色实心样式（`background: var(--color-accent); color: #fff;`），不可操作时切换为灰底灰字禁用态。所有可执行操作必须有 ready/disabled 两种状态的视觉区分，不能让按钮回归浏览器默认样式或使用过于低调的 ghost 按钮。次要操作（出售、转移等）可以较小，但仍需保持与主操作一致的视觉语言。

### Pixel Icon Design System

#### Style
**水墨像素（Ink-Wash Pixel）** — 像素风格但使用项目现有水墨配色（宣纸底、赭石色、墨灰色），与 UI 融为一体。不使用鲜艳复古色彩，保持禅意克制。

#### Technical Approach
- **格式**：React SVG 组件（inline SVG, 24×24 viewBox）
- **像素风格**：使用 `shape-rendering: crispEdges` + `image-rendering: pixelated` 确保像素锐利
- **每像素尺寸**：24px viewBox 内用 2px 为一个像素单位 → 实际 12×12 像素格

#### Icon Categories & Priority
1. **建筑类（8个）**：宗门大殿、灵石矿、灵田、坊市、丹炉、炼器坊、藏经阁、聚仙台
2. **弟子/角色类**：境界图标（6个）、状态图标（8个）、品质图标（5个）
3. **装备/物品类**：装备槽位图标（9个）、物品类型图标（4个）、消耗品图标（5个）
4. **元素/技能类**：元素图标（5个）、技能类型图标（4个）
5. **系统 UI 类**：资源图标（4个）、操作图标（关闭/确认/筛选等）
