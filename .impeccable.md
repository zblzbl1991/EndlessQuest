## 设计上下文

### 用户
仙侠放置游戏爱好者与 Roguelike 策略玩家的交集人群。他们熟悉修仙放置/挂机类手游，也喜欢以撒、杀戮尖塔等 Roguelike 的策略深度。主要在碎片时间和晚间放松时段游玩，希望用很少的操作完成真实的经营判断、弟子培养和单局冒险押注。

### 品牌气质
三个关键词：**古朴、禅意、有力量感**

- **古朴**：整体视觉扎根于传统东方美学，强调水墨、宣纸、书法等元素
- **禅意**：大量留白、舒缓节奏、不喧闹的界面
- **有力量感**：平时克制，关键节点必须有浓墨重彩的压迫感和仪式感

### 美术方向
- **视觉方向**：浅色水墨风格 + 仙侠手游 UI 布局
- **参考**：修仙放置类手游的仪表布局和信息组织，但表达更克制、更像一幅会呼吸的宗门界面，而不是后台面板
- **反参考**：避免 MMO 式重度 UI（太多按钮/弹窗/数字堆砌/满屏发光特效），也避免用任务清单或强提示直接告诉玩家该做什么
- **主题**：浅色模式。日常壳层以 `清墨轻岚` 为基底：冷宣纸白、雾青灰、浅墨蓝构成清淡素雅的日常界面；赭石 / 印泥色只保留给关键主操作、完成反馈和里程碑高光，不能重新把整个产品拉回暖褐旧卷
- **布局**：移动端底部标签栏 + 桌面端左侧侧边栏，CSS 响应式切换，无需 JS 检测设备类型；`640-1023px` 的平板区间必须被单独设计，不能只是更宽的手机页面
- **界面气质**：首页更像轻提示总览，展示宗门正在发生的变化，不替用户排优先级
- **语汇统一**：图标、按钮、标签、导航必须来自同一套水墨世界观，避免表情符号、通用后台图标和像素水墨语言混用；也避免脱离主题的蓝紫色后备颜色混入

### 响应式策略

移动端与 Web 端双端适配。移动端为主（核心体验），Web 端充分利用更宽屏幕提升信息密度和操作效率。

- **断点**：`640px`（移动端上限 / 平板起点）、`1024px`（平板上限 / 桌面起点）
- **移动端（< 640px）**：底部标签导航、单列布局、底部抽屉弹窗、44px 最小触控区域
- **平板端（640-1023px）**：继续使用底部标签导航，但内容宽度必须放宽到平板阅读尺度；页面主视觉、摘要、筛选、列表至少进入稳定的双列或主次并排结构；弹层优先使用底部抽屉或贴近底边的轻抽屉，而不是照搬桌面居中浮层
- **桌面端（≥ 1024px）**：左侧固定侧边栏导航（收起时仅图标，展开时图标+文字）、多列网格布局、面板可并排、弹窗居中对话框；宽屏空间优先用于持续可见的宗门上下文、摘要与并排面板，而不是简单居中放大移动端单列
- **导航切换**：桌面侧边栏与移动端底部导航通过同一组件 + CSS 响应式切换，不依赖 JS 检测
- **参考风格**：修仙手游 PC 模拟器体验的原生化适配（非简单居中模拟）

### 设计原则

1. **留白即呼吸**
   水墨画的灵魂在于留白。UI 不是把所有信息塞满屏幕，而是通过充足的间距和留白让界面"透气"。每个面板、每个区块之间要有呼吸感。信息密度适度，宁可多一层点击也不要一次展示太多。

2. **水墨晕染，不张扬**
   视觉效果以水墨质感为主——渐变、晕染、淡入淡出。特效（战斗、突破）可以用浓墨泼洒，但日常界面保持清淡素雅。避免霓虹发光、厚重的描边、过度饱和的颜色。

3. **关键时刻，浓墨重彩**
   日常界面保持禅意和克制，但境界突破、通关秘境、获得稀有物品等成就时刻必须有强烈的视觉反馈。用浓墨泼溅、金色光效、大幅动画来制造仪式感和成就感。反差越大，冲击越强。

4. **信息层级如山水远近**
   像山水画有近景中景远景一样，UI 有清晰的层级关系。最重要的信息最近最清晰，次要信息渐远渐淡。避免所有卡片同重、所有标题同亮、所有说明字同密度的平铺感。

5. **少指挥，多呈现**
   设计要呈现变化和风险，而不是替玩家下判断。首页、战报、弟子页都应提供判断素材，不应做成任务清单、系统推荐或过强的优先级指挥。

6. **宗门先于工具**
   导航、页头、面板不只是功能容器，还要让用户感到自己身处宗门。避免后台系统感过强的壳子，减少临时占位式图标、表情符号和纯管理台式分区。

7. **可操作即可见**
   当玩家可以对某物执行操作时（建造、升级、锻造、参悟等），必须有明确的按钮视觉提示。主操作按钮统一使用明确的主强调色实心样式（`background: var(--color-accent); color: #fff;`），不可操作时切换为灰底灰字禁用态。所有可执行操作必须有 ready/disabled 两种状态的视觉区分，不能让按钮回归浏览器默认样式或使用过于低调的 ghost 按钮。次要操作（出售、转移等）可以较小，但仍需保持与主操作一致的视觉语言。

8. **日常克制，高光放大**
   日常页面可以素、静、轻，但突破、稀有掉落、重要通关、命运变化等高光瞬间必须显著抬高视觉反差。没有高光反差，整个产品会失去记忆点。

9. **平板不是放大的手机**
   `640-1023px` 不是过渡废区。平板端应保留移动端触控逻辑，但在信息编排上显著增强：更宽的内容容器、更多并排关系、更稳定的摘要和筛选布局。不要把 `max-width: 480px` 的手机页面直接摆在平板中央。

10. **色彩分工要清楚**
   日常页面以清墨轻岚为基底；强调色只负责主操作、当前态和重要结论；赭石 / 印泥色只给关键节点。不要让日常边框、普通说明、主按钮和高光场景共用同一强度的颜色，也不要用主题外的通用蓝紫色补洞。

### 像素图标系统

#### 风格
**水墨像素** — 像素风格但使用项目现有水墨配色（冷宣纸白、浅墨蓝、雾青灰、受控赭石高光），与界面融为一体。不使用鲜艳复古色彩，保持禅意克制。

#### 技术实现
- **格式**：React SVG 组件（inline SVG, 24×24 viewBox）
- **像素风格**：使用 `shape-rendering: crispEdges` + `image-rendering: pixelated` 确保像素锐利
- **每像素尺寸**：24px viewBox 内用 2px 为一个像素单位 → 实际 12×12 像素格

#### 调色板（来自 `theme.css` 变量）
| 用途 | 当前基线 | 变量 |
|---|---|---|
| 日常主强调 | 清墨蓝灰 | `--color-accent` |
| 主文字 | 深墨灰 | `--color-text` |
| 次文字 | 雾青灰 | `--color-text-secondary` |
| 弱文字 / 禁用 | 纸雾灰 | `--color-text-tertiary` |
| 页面底色 | 冷宣纸白 | `--color-bg` |
| 面板纸色 | 轻纸白 | `--color-bg-paper` |
| 品质-凡 | 灰阶 | `--color-quality-common` |
| 品质-灵 | 冷灵蓝 | `--color-quality-spirit` |
| 品质-仙 | 淡仙紫 | `--color-quality-immortal` |
| 品质-神 | 受控金赭 | `--color-quality-divine` |
| 品质-混沌 | 赤墨红 | `--color-quality-chaos` |

图标在日常页面优先使用清墨轻岚基线；只有关键高光、稀有品质或里程碑语境下，才允许少量引入更重的赭石 / 印泥感强调。

#### 图标分类与优先级
1. **建筑类（8个）**：宗门大殿、灵石矿、灵田、坊市、丹炉、炼器坊、藏经阁、聚仙台
2. **弟子/角色类**：境界图标（6个）、状态图标（8个）、品质图标（5个）
3. **装备/物品类**：装备槽位图标（9个）、物品类型图标（4个）、消耗品图标（5个）
4. **元素/技能类**：元素图标（5个）、技能类型图标（4个）
5. **系统界面类**：资源图标（4个）、操作图标（关闭/确认/筛选等）
